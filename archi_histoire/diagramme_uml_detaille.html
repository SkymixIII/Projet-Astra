Architecture du jeu basee sur archi.md — toutes les classes, interfaces, enums et leurs relations.

1. Hierarchie : Item / Batiment / Joueur / Ouvrier

L'interface Item est la racine de tout element positionne sur la carte. Batiment en herite avec une position fixe. Joueur et Ouvrier sont mobiles.

classDiagram
    class Item {
        <<interface>>
        +int x
        +int y
        +distance(Item) double
        +deplacer(int x, int y) void
        +deplacer(Direction d) void
    }
    class Direction {
        <<enumeration>>
        NORD
        SUD
        EST
        OUEST
    }
    class Batiment {
        <<interface>>
        +int sante
        +List~Ouvrier~ personnelAffecte
        +int maxPersonnel
        +isOperationnel() boolean
        +affecterPersonnel(Ouvrier) void
        +retirerPersonnel(Ouvrier) void
        +subirDegats(int) void
        +reparer(int) void
    }
    class Joueur {
        -Direction orientation
        -Map~TypeRessource_Integer~ ressources
        -List~Ouvrier~ ouvriers
        -List~Batiment~ batiments
        +creerJoueur() void
        +deplacer(int x, int y) void
        +deplacer(Direction) void
    }
    class Ouvrier {
        -int identifiant
        -Role role
        -int faim
        -int soif
        -int repos
        -int moral
        -int experience
        -EtatOuvrier etat
        -boolean disponibilite
        +gagnerExp(int) void
        +getNiveau() int
        +evoluer() void
        +affecterPoste(Batiment) void
        +mettreAJourBesoins() void
        +getProductivite() float
    }
    Batiment --|> Item : extends
    Joueur ..|> Item : implements
    Ouvrier ..|> Item : implements
    Joueur --> Direction : orientation
Note : Batiment herite de Item mais sa position (x, y) est fixee a la construction. Joueur et Ouvrier implementent Item et sont mobiles (methodes deplacer).

Transport : Pour les ouvriers, pas de calcul de trajet (les ressources vont directement au stock). Le CalculateurTrajet est utilise seulement pour le joueur.
RessourceInsuffisanteException : Lancee par Usine.produire() quand le stock est insuffisant. Attrapee par la boucle du jeu.
RechercheImpossibleException : Lancee par ArbreTechnologique.lancerRecherche() quand les prerequis ne sont pas remplis.
